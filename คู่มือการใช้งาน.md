# คู่มือการใช้งาน CCTV Automation

ระบบแคปภาพกล้องวงจรปิดภายในเครื่อง พร้อมหน้าเว็บสำหรับบันทึกภาพสด บันทึกข้ามคืน และดึงภาพย้อนหลังจาก NVR

เว็บภายในเครื่อง: `http://127.0.0.1:8787/`

## สิ่งที่โปรเจกต์ทำได้

- แคปภาพรวมจากกล้องที่ตั้งไว้ทั้งหมด 18 ช่อง
- สร้างภาพรวมอัตโนมัติ และเก็บภาพกล้องแยกแต่ละช่อง
- เลือกกล้องที่จะคัดลอกไปไว้ในโฟลเดอร์ `รูปเดี่ยว`
- บันทึกข้ามคืนตามเวลาที่กำหนด เช่น `23:00` และ `04:00`
- บันทึกย้อนหลังแบบข้ามคืน โดยใช้วันที่เริ่มต้นและวันที่ถัดไป
- แสดงผลภาพในเว็บ เปิดดู หรือดาวน์โหลดได้
- ยกเลิกงานที่กำลังทำอยู่ได้

## ไฟล์ `.env` ต้องขอจากใคร

ให้ขอไฟล์ `.env` จาก **พี่หมี** เพราะไฟล์นี้มีข้อมูลสำหรับเชื่อมต่อ NVR เช่น Host, Username และ Password

ห้ามสร้างค่าเองจาก `.env.example` หากยังไม่มีข้อมูล NVR และห้ามส่งไฟล์ `.env` เข้า GitHub หรือแชตสาธารณะ

## ต้องวางไฟล์ `.env` ไว้ที่ไหน

วางไฟล์ชื่อ `.env` ไว้ใน **โฟลเดอร์หลักของโปรเจกต์** ระดับเดียวกับ `app.py`

โครงสร้างที่ถูกต้อง:

```text
CCTV Autometion/
├── app.py
├── .env                    ← วางไฟล์จากพี่หมีตรงนี้
├── .env.example
├── เปิดโปรแกรม.command
├── ติดตั้งระบบ.command
├── install_windows.bat
├── เปิดโปรแกรม_windows.bat
├── static/
└── captures/
```

ห้ามวาง `.env` ไว้ใน `static/` หรือ `captures/` และตรวจสอบว่าไฟล์ชื่อ `.env` จริง ๆ ไม่ใช่ `.env.txt`

### ตำแหน่งบน macOS เครื่องนี้

```text
/Volumes/DriveG/dev/workspace/github/CCTV Autometion/.env
```

### ตำแหน่งบน Windows

วางไว้ในโฟลเดอร์ที่เก็บโปรเจกต์ เช่น:

```text
C:\CCTV Autometion\.env
```

## โปรแกรมที่ต้องติดตั้ง

จำเป็น:

- Python 3 สำหรับรันเว็บเซิร์ฟเวอร์
- ffmpeg สำหรับอ่าน RTSP และสร้างภาพรวม

Node.js **ไม่จำเป็น** สำหรับโปรเจกต์เวอร์ชันปัจจุบัน เพราะหน้าเว็บใช้ HTML/JavaScript และ Tailwind CDN โดยตรง

## ติดตั้งบน macOS

วิธีง่ายที่สุด ดับเบิลคลิกไฟล์:

```text
ติดตั้งระบบ.command
```

หรือใช้ Homebrew:

```bash
brew install python ffmpeg
```

จากนั้น:

1. ขอไฟล์ `.env` จากพี่หมี
2. วาง `.env` ไว้ข้าง `app.py`
3. ดับเบิลคลิก `เปิดโปรแกรม.command`
4. เปิดเว็บที่ `http://127.0.0.1:8787/`

ถ้าต้องการรันด้วยคำสั่ง:

```bash
cd "/Volumes/DriveG/dev/workspace/github/CCTV Autometion"
python3 app.py
```

## ติดตั้งบน Windows

ถ้าเครื่องมี `winget` ให้ดับเบิลคลิก:

```text
install_windows.bat
```

หรือเปิด PowerShell/Command Prompt แล้วใช้:

```bat
winget install --id Python.Python.3.12 --exact --source winget
winget install --id Gyan.FFmpeg --exact --source winget
```

หลังติดตั้ง ให้ปิดแล้วเปิดหน้าต่าง Terminal ใหม่ จากนั้น:

1. ขอไฟล์ `.env` จากพี่หมี
2. วาง `.env` ไว้ในโฟลเดอร์เดียวกับ `app.py`
3. ดับเบิลคลิก `เปิดโปรแกรม_windows.bat`
4. เปิดเว็บที่ `http://127.0.0.1:8787/`

Node.js เป็นตัวเลือกเท่านั้น หากต้องการพัฒนาหน้าเว็บด้วย React ในอนาคต:

```bat
winget install --id OpenJS.NodeJS.LTS --exact --source winget
```

## วิธีใช้งานหน้าเว็บ

### บันทึกข้ามคืน

1. เลือกแท็บ **บันทึกข้ามคืน**
2. ตั้งเวลาแคป 2 รอบ เช่น `23:00` และ `04:00`
3. เลือกกล้องสำหรับรูปเดี่ยว
4. กด **บันทึกเวลา**
5. กด **เปิดรันข้ามคืน**
6. เปิดเครื่องและโปรแกรมทิ้งไว้จนกว่างานจะเสร็จ

### บันทึกย้อนหลัง

1. เลือกแท็บ **บันทึกย้อนหลัง**
2. เลือกวันที่เริ่มต้น
3. รอบแรกจะเป็นวันที่เลือก เช่น `23:00`
4. รอบที่สองจะเป็นวันถัดไป เช่น `04:00`
5. เลือกช่วงเวลาและกล้องรูปเดี่ยว
6. กด **บันทึกย้อนหลัง 2 ช่วงเวลา**
7. ดูความคืบหน้าและภาพได้ที่หัวข้อ **ผลลัพธ์ภาพ**

หากงานใช้เวลานานหรือสั่งผิด ให้กด **ยกเลิกงาน**

## ตำแหน่งผลลัพธ์

```text
captures/YYYY-MM-DD/HH-MM/
├── รวมกล้อง/
│   ├── camera-01_....jpg
│   └── ภาพรวม-18-ช่อง_....jpg
├── รูปเดี่ยว/
│   └── camera-01_....jpg
└── manifest.json
```

ถ้าบางกล้องไม่มีไฟล์ย้อนหลังหรือ Playback ตอบไม่ทัน ระบบจะแสดงเป็น **เสร็จบางส่วน** พร้อมระบุหมายเลขกล้องที่มีปัญหา ภาพจากกล้องอื่นยังใช้งานได้

## แก้ปัญหาเบื้องต้น

- เชื่อมต่อ NVR ไม่ได้: ตรวจสอบเครือข่ายและข้อมูลใน `.env`
- ไม่พบ ffmpeg: ติดตั้ง ffmpeg ใหม่ แล้วเปิด Terminal/Command Prompt ใหม่
- ได้ภาพไม่ครบ: ดูหมายเลขกล้องที่แจ้งเตือน และตรวจ Playback ของกล้องนั้นที่ NVR
- ดูรายละเอียดเพิ่มเติมได้ที่ `data/app.log`

## ความปลอดภัย

ไฟล์ `.env` มีข้อมูลสำหรับเข้าถึงภาพกล้อง เก็บไว้ในเครื่องที่ได้รับอนุญาตเท่านั้น และไม่ควรแนบไฟล์นี้ไปกับเอกสารหรืออัปโหลดสาธารณะ
